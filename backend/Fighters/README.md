# Gladiators
## Требования
Доработать консольную игру, в которой все игровые персонажи должны реализовывать интерфейс IFighter. Интерфейс IFighter должен содержать методы для атаки и проверки состояния бойца. После создания персонажей происходит схватка, победитель – последний живой боец.
 
### Персонаж
Каждый персонаж в игре должен иметь уникальные характеристики, такие как здоровье, базовая сила и защита. У персонажа есть раса, оружие, броня и класс (“наемник”, “рыцарь”...)
 
### Характеристики персонажа
- Все статы увеличивают макс. ХП
- Сила => увеличивает макс. ХП  сильнее всего, увеличивает на 1% максимальное значение раброса наносимого урона
- Ловкость => влияет на инициативу бойца, чем больше тем лучше
- Интиллект => увеличивает шанс крита

### Характеристики оружия
- Урон - имеет разброс, шанс и силу крита, тип урона (физ, маг, чистый, яд)
- Статы оружия - максимальный предел, сколько статов бойца может использовать оружие
    Пример: у бойца 60 силы, предел силы у оружия 50. Тогда бонус к урону будет = 50 * 5% + (60 - 50) * 0.5% = 260% усиления

### Характеристики брони
- Сопротивления урону в процентах
- Уникальные эффекты
 
### Создание персонажа
- Приложение позволяет делать поэтапную конфигурацию персонажа.
- На каждого бойца выделяется N очков, их можно потратить на статы персонажа (1 пункт стата = 1 очко).
- Оставшиеся очки можно потратить на броню и оружие.
- Раса и класс выдаются бесплатно.
 
### Битва
Персонажи сталкиваются в пораундовой битве с выводом в консоль результатов (“Персонаж А получил 10 урона, нанес 20”). Выигрывает последний оставшийся персонаж (“Персонаж Ц выжил и победил”)
